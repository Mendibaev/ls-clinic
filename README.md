# LS Clinic — сайт медицинского центра (Алматы)

Прототип сайта многопрофильной клиники LS Clinic. Чистый front-end без сборки и зависимостей — один HTML-файл со встроенными стилями и скриптами. Адаптивная вёрстка (mobile-first), полная мультиязычность RU/KZ.

## Демо

Открыть `index.html` в браузере. Для публикации онлайн подойдёт GitHub Pages (см. ниже).

## Возможности

- **Мультиязычность RU/KZ** — переключатель в шапке, перевод навигации, заголовков, кнопок, карточек врачей, прайса и раздела ОСМС.
- **Hero-слайдер** с 6 ключевыми предложениями (автопрокрутка, стрелки, точки).
- **Разделение потоков пациентов** — платные услуги (красная стилистика) и ОСМС / прикрепление (зелёная).
- **Платные направления** — хирургия, материнство и детство, lifestyle-медицина, B2B и справки.
- **Раздел ОСМС** — инструкция по онлайн-прикреплению через eGov, список бесплатных услуг, блок вопросов и ответов, блог главврача.
- **Врачи** — сетка карточек с переходом на отдельную страницу-профиль каждого специалиста (биография, образование, что лечит, стоимость приёма).
- **Интерактивный прайс** — поиск по услугам, фильтр по категориям (консультации / анализы / операции / программы), быстрый заказ звонка.
- **Сбор лидов** — модальная форма записи (имя + телефон) и форма «Письмо директору».
- **Валидация форм** — маска телефона `+7 (XXX) XXX-XX-XX`, проверка имени и сообщения, подсветка ошибок.
- **Плавающая кнопка WhatsApp.**

## Локальный запуск

Достаточно открыть файл двойным кликом. Либо поднять простой сервер:

```bash
# Python 3
python3 -m http.server 8000
# затем открыть http://localhost:8000
```

## Публикация на GitHub Pages

```bash
git init
git add .
git commit -m "LS Clinic: первичный прототип сайта"
git branch -M main
git remote add origin https://github.com/<USERNAME>/ls-clinic.git
git push -u origin main
```

Затем в настройках репозитория: **Settings → Pages → Source: branch `main`, папка `/ (root)`**. Сайт будет доступен по адресу `https://<USERNAME>.github.io/ls-clinic/`.

## Технологии

Чистые HTML5, CSS3 и ванильный JavaScript. Внешних библиотек и шагов сборки нет.

## Дальнейшие шаги

Прототип статический. Для продакшена потребуется:

- подключить отправку заявок на почту администрации и в amoCRM;
- интегрировать виджет / инструкцию прикрепления eGov с реальными данными;
- заменить тексты, цены и фотографии врачей на актуальные;
- при необходимости — вынести страницы в отдельные файлы или перевести на CMS / фреймворк.
