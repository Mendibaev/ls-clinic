# LS Clinic — сайт медицинского центра (Алматы)

Сайт многопрофильной клиники LS Clinic. Статический front-end без фреймворков. Каждая страница — отдельный файл со своим «чистым» URL (как у новостных порталов), адаптивная вёрстка (mobile-first), мультиязычность RU/KZ с запоминанием выбора.

## Чистые URL

| Страница | URL |
| --- | --- |
| Главная | `/` |
| Платные услуги | `/surgery/` |
| ОСМС и прикрепление | `/osms/` |
| Врачи | `/doctors/` |
| Профиль врача | `/doctors/<имя-врача>/` (8 страниц) |
| Прайс | `/price/` |
| Письмо директору | `/director/` |

## Структура

```
.
├── index.html                  # главная
├── surgery/index.html          # платные услуги
├── osms/index.html             # ОСМС
├── doctors/index.html          # список врачей
│   └── <slug>/index.html       # профиль врача (×8)
├── price/index.html            # прайс
├── director/index.html         # письмо директору
├── assets/
│   ├── style.css               # все стили
│   └── app.js                  # данные, i18n, логика (общий для всех страниц)
├── build.js                    # генератор страниц (см. ниже)
├── README.md
├── .gitignore
└── LICENSE
```

Общие шапка, подвал, модальная форма и стили не дублируются вручную: HTML-страницы генерируются скриптом `build.js` из единых шаблонов, а вся интерактивность и данные лежат в одном `assets/app.js`.

## Как вносить правки

Тексты блоков правятся прямо в HTML-файлах (или в шаблонах внутри `build.js`). Данные врачей, прайс, переводы RU/KZ — в `assets/app.js`. После изменения шаблонов/контента в `build.js` перегенерируйте страницы:

```bash
node build.js
```

Если правите только стили или логику — меняйте `assets/style.css` / `assets/app.js`, перегенерация не нужна.

## Локальный запуск

Откройте `index.html` или поднимите простой сервер (нужно для корректной работы относительных путей между папками):

```bash
python3 -m http.server 8000
# http://localhost:8000
```

## Возможности

- Чистые URL для каждой страницы — удобно для SEO и для ссылок.
- Мультиязычность RU/KZ с сохранением выбора в localStorage между страницами.
- Hero-слайдер на главной, разделение потоков (платные / ОСМС).
- Платные направления, раздел ОСМС (инструкция eGov, бесплатные услуги, Q&A, блог).
- Врачи: список + отдельная страница-профиль каждого специалиста.
- Интерактивный прайс с поиском и фильтром по категориям.
- Сбор лидов: модальная форма записи и форма «Письмо директору».
- Валидация форм, маска телефона `+7 (XXX) XXX-XX-XX`.
- Плавающая кнопка WhatsApp.

## Публикация на GitHub Pages

```bash
git init
git add .
git commit -m "LS Clinic: многостраничный сайт с чистыми URL"
git branch -M main
git remote add origin https://github.com/<USERNAME>/ls-clinic.git
git push -u origin main
```

Затем **Settings → Pages → Source: branch `main`, папка `/ (root)`**. Папочная структура (`/surgery/index.html` и т.д.) даёт чистые адреса автоматически, без серверных правил.

## Технологии

HTML5, CSS3, ванильный JavaScript. Node.js используется только как генератор страниц (`build.js`) на этапе разработки; готовые файлы работают как обычная статика.

## Дальнейшие шаги

Прототип статический. Для продакшена: подключить отправку заявок на почту и в amoCRM, интегрировать данные eGov, заменить тексты/цены/фото врачей на актуальные.
