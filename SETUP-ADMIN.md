# Админка и внешний контент (Sanity)

Весь контент сайта теперь может загружаться из Sanity — облачной CMS с готовой
визуальной админкой. Пока Sanity не настроен, сайт продолжает работать на
встроенных данных (`src/data/*`) — ничего не ломается.

## Как это устроено

- **Админка** — Sanity Studio (отдельное приложение в этом же репозитории,
  папка `sanity/` + `sanity.config.js`). Запускается локально (`npm run studio`)
  и деплоится в облако одной командой.
- **Сайт** при старте делает один запрос к Sanity и подставляет контент.
  Если Sanity недоступен/не настроен — используются встроенные данные (фолбэк).
- **Сидовые данные** (`src/data/*`) остаются как резервная копия и как источник
  для первичного импорта.

## Разовая настройка (≈10 минут)

1. **Создайте проект Sanity** (бесплатно): https://www.sanity.io → Sign up →
   создайте проект, dataset назовите `production`. Запишите **Project ID**.

2. **Заполните `.env`** в корне проекта:
   ```
   cp .env.example .env
   ```
   Впишите:
   - `VITE_SANITY_PROJECT_ID` и `SANITY_STUDIO_PROJECT_ID` — ваш Project ID
   - `SANITY_API_TOKEN` — токен с правами записи: sanity.io → проект → **API →
     Tokens → Add token**, роль **Editor**. Нужен только для импорта.

   > `.env` уже в `.gitignore` — токен в репозиторий не попадёт.
   > Project ID не секретный; при желании можно вписать его прямо в
   > `sanity/projectConfig.js`.

3. **Установите зависимости** (добавились пакеты Sanity):
   ```
   npm install
   ```

4. **Перенесите текущий контент в Sanity** (разово):
   ```
   npm run import
   ```
   Скрипт загрузит филиалы, направления, врачей, услуги, анализы, акции, блог,
   вакансии, отзывы и настройки. Повторный запуск безопасен (перезапись, без
   дубликатов).

5. **Разрешите сайту читать данные.** В Sanity → **API → CORS origins**
   добавьте адреса сайта (например `http://localhost:5173` и ваш домен на
   Netlify). Dataset `production` по умолчанию публичный на чтение.

## Запуск

- Сайт: `npm run dev` → http://localhost:5173
- Админка локально: `npm run studio` → http://localhost:3333
- Опубликовать админку в облако: `npm run studio:deploy`
  → получите адрес вида `https://<имя>.sanity.studio`, где редакторы входят по
  логину (Google/GitHub/email). Доступы и роли добавляются в sanity.io →
  **Members** (легко расширяется на любое число редакторов).

## Деплой сайта (Netlify)

Добавьте переменные окружения в Netlify (Site settings → Environment variables):
`VITE_SANITY_PROJECT_ID` и `VITE_SANITY_DATASET=production`. Дальше обычный
деплой — контент будет подтягиваться из Sanity при загрузке страницы.

## Что редактируется в админке

Филиалы · Направления (с поднаправлениями и преимуществами) · Врачи · Услуги и
цены · Анализы · Акции · Блог/новости · Отзывы · ПМСП и ОСМС · Вакансии ·
Настройки сайта (телефоны, адреса, график, партнёры) · Страницы «О клинике»,
«Благотворительность», «Корпоративный отдел», «Политика конфиденциальности».

## Как добавить новое поле/тип

1. Опишите поле в `sanity/schemaTypes/index.js`.
2. Добавьте его в GROQ-запрос `src/content/queries.js`.
3. Примените в `src/content/store.js` (маппинг на нужный массив/объект).
